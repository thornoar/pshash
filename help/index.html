Help information

The work of the pshash pseudo-hash function is pictured here:

The first stage is the Merge-Choice stage (MC), which takes a source configuration L and a key k1 (the private choice key), picking the given amounts of elements from the sources and then shuffling them between each other. The second, Shuffle stage takes the resulting list of elements and a second key k2 (the private shuffle key), performing a second shuffle on the list. This is done to ensure that multiple pairs of keys return the same funal pseudo-hash, and thus the user's particular pair is impossible to guess.

The pshash function accepts the following inputs:

  • A source configuration, that is, a set of data that specifies what strings to pick characters from and how many to pick. There are predefined constants available under the keywords:
    • long (8 upper case, 8 lower case, 5 special characters, 4 digits). This is the default configuration.
    • medium (5 symbols of each type)
    • short (4 symbols of each type)
    • anlong (7 upper case, 7 lower case, 7 digits)
    • anshort (4 upper case, 4 lower case, 4 digits)
    • pin (4-digit pin code)
    • mediumpin (6-digit pin code)
    • longpin (8-digit pin code)
  • A public key, i.e. a string of characters representing the website or service that requires the password. Examples include "google", "steam", "spotify", etc.
  • A private choice key, a number between 0 and 10^42 (upper bound may depend on the source configuration).
  • A private shuffle key, a number between 0 and 10^25 (upper bound may depend on the source configuration).
